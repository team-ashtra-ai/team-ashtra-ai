Accessibility

Built to be clearer. Built to be usable.

ASH-TRA aims to make ash-tra.com easier to use for more people by following practical accessibility best practices and improving the experience over time.

Clarity is part of the standard.

Readable Clear structure, sensible contrast, and readable text.
Navigable Keyboard-friendly use, focus visibility, and reduced friction.
Responsive Cleaner interactions across devices and screen sizes.
Improving Accessibility is treated as an ongoing process, not a checkbox.
Accessibility at a glance

Readable, navigable, responsive, improving.

ASH-TRA treats accessibility as an ongoing standard built around clear structure, usable interactions, and practical support when issues appear.

  • Structure, contrast, readability, and keyboard use are part of the baseline.
  • The site should continue improving as pages and features evolve.
  • Third-party tools may behave differently from the main site.
  • Accessibility issues can be reported directly by email.

Feedback: team.ashtra.ai@gmail.com

1. Approach

ASH-TRA aims to support:

  • clear structure
  • readable text
  • sensible contrast
  • cleaner navigation
  • reduced friction across devices
  • keyboard-friendly use where possible
  • more usable interactions overall

2. Ongoing improvement

Accessibility is treated as an ongoing process, not a one-time checkbox. The site may continue to evolve as features change, pages grow, and improvements are made.

3. Third-party tools

Some third-party tools, embeds, booking systems, payment tools, or integrations may behave differently from the rest of the site. ASH-TRA aims to choose these carefully, but cannot fully control every external service.

4. Feedback

If you encounter an accessibility issue or need support accessing any part of the site, contact team.ashtra.ai@gmail.com.

Please include:

  • the page involved
  • the issue you found
  • the help you need
