# Content Semantics

## Heading Rules

- Use exactly one semantic `h1` per page.
- Use `h2` through `h6` only when the document outline actually needs them.
- Use display utility classes for oversized treatments instead of inventing `h7` or `h8`.

## Paragraph Rules

Use the shared paragraph classes instead of inventing one-off text sizing:

- lead
- body
- body-small
- caption
- meta

The goal is consistent readability, line length, and spacing across public and dashboard surfaces.

## Badge Copy

- Prefer short noun phrases.
- Avoid repeating the surrounding heading text.
- Use badges to create scan rhythm, not to inflate density.

## Image Alt Text

- Describe the scene plainly.
- Tie it to the page purpose when helpful.
- Do not stuff keywords.
- Do not write placeholder alt text like "marketing image" or "stock photo."

## SEO Metadata

Each public page should keep:

- one focused title
- one focused description
- aligned OG/Twitter text
- structured data only where it adds clarity
